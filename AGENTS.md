# AirFit Agent Guide

Use this file as the first repo-specific context for future Codex sessions. Keep it concise and update it when repeated mistakes or new workflows appear.

## Project Snapshot

AirFit is a Create React App single-page app for a gym client/admin portal. The frontend is hosted on Vercel at `https://airfitworkout.vercel.app`. The backend is not in this repo; it is a set of n8n workflows exposed through webhooks under `REACT_APP_N8N_BASE`.

Active runtime path:

- `public/index.html`
- `src/index.js`
- `src/App.js`

Do not use `src/main.jsx` or `src/App.jsx`; those files were removed as stale alternate entry points.

## Important Directories

- `src/pages/` - route-level screens for admin and client flows.
- `src/components/` - shared UI components such as questionnaire, plan cards, charts, calorie tracker, and admin table.
- `src/hooks/` - auth, plan polling, and progress state hooks.
- `src/utils/` - endpoint config, localStorage helpers, invite client, and plan parsing.
- `public/` - static assets served by CRA.
- `wf_*.json` - exported n8n workflows for reference only; live workflow edits happen in n8n, not through the React app.
- `build/` - generated by `npm run build`; do not edit or commit.

## Commands

Only document commands that exist or are directly supported by the current CRA setup.

- Install dependencies: `npm install`
- Start local dev server: `npm start`
- Production build: `npm run build`
- Jest watch mode: `npm test`
- One-shot tests for verification: `npm test -- --watchAll=false`

There are no dedicated `lint`, `format`, or `typecheck` scripts in `package.json`. The CRA production build runs ESLint checks as part of `npm run build`. This is a JavaScript app; there is no TypeScript typecheck command.

## Environment

Use `.env.example` as the template. Required/known variables:

- `REACT_APP_URL` - public app origin used for invite links when set.
- `REACT_APP_N8N_BASE` - n8n webhook base URL, for example `https://airfitplangen.duckdns.org/webhook`.
- `REACT_APP_ADMIN_PASSWORD` - admin login passcode.
- `REACT_APP_API_TOKEN` - optional token used by questionnaire submission when configured.

Important: `.env` and `.env.production` are currently tracked in this repo history and contain real values. Do not print, stage, rotate, or edit them unless the user explicitly asks. Prefer Vercel project environment variables for production secrets.

## Routes And Flows

Routes are defined in `src/App.js`:

- `/` redirects to `/client/login`
- `/admin/login`
- `/admin/dashboard`
- `/client/login`
- `/client/dashboard`
- `/client/waiting`
- `/client/plan`
- `/client/progress`

Auth is client-side:

- Admin session key: `airfit_admin_session`
- Client session key: `airfit_client_session`
- Client login verifies phone numbers through n8n.
- Workout progress and calorie logs are stored in browser `localStorage`.

## API And Backend

Endpoints are centralized in `src/utils/config.js`:

- `airfit-gym-diet` - submit questionnaire for plan generation.
- `airfit-get-plan` - poll plan status and retrieve generated plan data.
- `airfit-plan-ready` - plan-store save endpoint used by n8n.
- `airfit-invite` - send portal invite email.
- `airfit-get-clients` - admin client list.
- `airfit-verify-client` - client lookup by phone or ID.
- `airfit-register-client` - register a client in n8n.

The exported workflow files map to:

- `wf_5WJMgjRnuIJVdA3h.json` - Portal Invite Email.
- `wf_maiz82IchPuO1an9.json` - Plan Generator.
- `wf_N7ujfqztm0xMe1Iu.json` - Plan Store.

Do not assume the JSON exports prove the live n8n workflows are current. Verify live behavior with safe webhook checks when a task depends on backend behavior.

## Conventions And Constraints

- Keep application behavior unchanged unless the user explicitly asks for a product/code change.
- Prefer existing React/CRA patterns and inline-style conventions already present in the app.
- Do not introduce a new framework, router, state manager, formatter, linter, or test runner without explicit approval.
- Keep endpoint definitions centralized in `src/utils/config.js`.
- Keep localStorage key compatibility unless a migration is part of the task.
- Treat `dangerouslySetInnerHTML` diet-plan rendering as an existing backend contract; do not broaden it casually.
- Avoid editing generated output in `build/`.
- Avoid committing local logs, secrets, `.vercel/`, `node_modules/`, or private MCP config.
- If testing n8n workflows, use clearly marked QA data and report any client IDs or phone numbers created.

## Verification Guidance

For documentation-only changes:

- Run `npm run build` if feasible to catch accidental code/doc import issues.
- Run `npm test -- --watchAll=false` when test behavior is relevant or when touching test files.

For app changes:

- Run `npm run build`.
- Exercise the affected route locally when possible.
- For backend-dependent flows, verify the relevant n8n endpoint with safe test data or report why network verification was not run.

For deployment tasks:

- Confirm the Vercel account/team and project before deploying.
- Avoid uploading private files; `.vercelignore` excludes local env and private config.
- After deployment, check `https://airfitworkout.vercel.app/client/login` and `https://airfitworkout.vercel.app/admin/login`.
